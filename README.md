## Завдання 2: Пошук шляхів у графі метро

### Реалізовані алгоритми

- **DFS (глибина-перший пошук):**
  Досліджує гілку графа максимально глибоко, перш ніж повернутися назад. Знайдений шлях може бути не найкоротшим за кількістю ребер.

- **BFS (пошук у ширину):**
  Досліджує всіх сусідів на поточному рівні, перш ніж переходити на наступний. Завжди знаходить найкоротший шлях (мінімальна кількість ребер) між двома вузлами.

### Результати та порівняння

- **BFS** завжди знаходить найкоротший маршрут від початкової до цільової станції.
- **DFS** може повертати довший або інший маршрут, залежно від порядку обходу сусідів.
- У даному графі метро BFS і DFS можуть давати різні шляхи між однією і тією ж парою станцій.

### Висновок

- Для знаходження найкоротшого шляху в невагованому графі краще використовувати BFS.
- DFS корисний для дослідження всіх можливих шляхів або у випадках, коли довжина шляху не є критичною.
